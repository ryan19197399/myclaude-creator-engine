# Documentation Standards

## README Requirements (MCS-1)
1. What it does (1-3 sentences)
2. How to install
3. How to use (with examples)
4. Requirements/dependencies

## Architecture Doc Requirements
1. System overview (1 paragraph + diagram)
2. Component inventory (name, purpose, location)
3. Data flow (input to processing to output)
4. Key decisions (and why)

## Quality Criteria
- Every code reference must point to an existing file
- No placeholder content
- Examples must be runnable
- Links must resolve
