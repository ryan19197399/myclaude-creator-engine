# Shared Vocabulary — MyClaude Creator Engine

> Canonical definitions used consistently across all agents, skills, and references.
> When in doubt about terminology, defer to this file.

---

## Core Concepts

| Term | Definition | Used By |
|------|-----------|---------|
| **Creator** | Any person using the Engine to create products for MyClaude | All agents, all skills |
| **Product** | Any artifact publishable on MyClaude marketplace. One of 9 categories. | All agents, all skills |
| **MCS** | MyClaude Creator Spec — 3-tier quality system (MCS-1: Publishable, MCS-2: Quality, MCS-3: State-of-the-Art) | quality-reviewer, validator |
| **Scaffold** | Auto-generated project structure for a product type, with guidance comments | scaffolder, domain-expert |
| **Guidance Comment** | `<!-- GUIDANCE: ... -->` comment in scaffold files that explains what to fill in. Stripped during /package (CE-D12) | scaffolder, publisher |
| **Workspace** | `workspace/` directory where products are actively built. Gitignored. One dir per product. | All skills |
| **Package** | Distribution-ready directory (.publish/) with product files + vault.yaml + README + LICENSE + CHANGELOG | publisher, packaging-specialist |
| **vault.yaml** | Machine-readable metadata file consumed by CLI and marketplace. Source of truth for product identity (CONDUIT WP-3, CE-D14). | publisher, packaging-specialist |
| **Activation Protocol** | Defined sequence of context-loading steps that a skill or agent executes before responding (CE-D34) | All agents, all skills |
| **Anti-Commodity Gate** | 3-question check that verifies a product has genuine human-injected differentiation (CE-D9). Coaches, never blocks (CE-D26). | quality-reviewer, differentiation-coach, validator |
| **Engine** | This system — the MyClaude Creator Engine itself | CLAUDE.md |
| **creator.yaml** | Adaptive profile of the creator's expertise, goals, and preferences. Generated by /onboard. | onboarder, all skills |
| **.engine-meta.yaml** | Per-product build state file in workspace. Tracks: created_at, last_validated, mcs_target, status, category. | scaffolder, validator, publisher |

## Quality Tiers

| Tier | Name | Meaning | Gate |
|------|------|---------|------|
| MCS-1 | Publishable | Minimum viable quality — structure, metadata, no broken refs | Automated |
| MCS-2 | Quality | Craft demonstrated — exemplars, anti-patterns, tested, no placeholders | Semi-automated + Anti-Commodity Gate |
| MCS-3 | State-of-the-Art | Expert validated — deep knowledge, composable, stress-tested, differentiated | Agent review (quality-reviewer) |

## Agent Handoff Protocol

When any agent hands off to another, the payload MUST include:

```yaml
handoff:
  from: "{agent-id}"
  to: "{target-agent-id}"
  product_path: "workspace/{slug}/"
  creator_profile: "{loaded from creator.yaml}"
  findings:
    - "{specific finding 1}"
    - "{specific finding 2}"
  recommendation: "{one-sentence recommended action}"
  context: "{why this handoff is happening}"
```

## Validation States

| State | Meaning | Set By |
|-------|---------|--------|
| `scaffolded` | Structure created, content not yet filled | scaffolder |
| `in-progress` | Creator is actively building content | creator (manual) |
| `validated` | Last validation run passed at target MCS level | validator |
| `packaged` | Product bundled into distribution ZIP | publisher |
| `published` | Product submitted to marketplace | publisher |
