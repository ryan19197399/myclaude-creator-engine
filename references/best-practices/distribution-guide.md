# Distribution Guide — Viral Growth Engineering

Every product created with MyClaude Studio Engine carries distribution DNA.
This guide explains the multi-channel strategy that makes every published product
a growth vector for both the creator AND the MyClaude ecosystem.

---

## The Viral Loop

```
Creator builds with Studio Engine
  -> Product carries MyClaude signature (README badge, manifests, metadata)
  -> Published to myclaude.sh (primary marketplace)
  -> Creator ALSO distributes to GitHub, npm, awesome lists, social
  -> Every distribution point links back to myclaude.sh
  -> Buyers discover MyClaude through the product
  -> Buyers become creators
  -> Repeat
```

**Key insight:** Every product is a marketing asset. The README is the most-viewed
file in the ecosystem. Badges, links, and attribution in READMEs create compound
backlinks across thousands of GitHub repos, npm packages, and social posts.

## Distribution Channels (ordered by impact)

### 1. MyClaude Marketplace (Primary)
- Automatic via `/publish`
- Full product page with reviews, MCS badge, install command
- SEO-optimized at myclaude.sh/p/{slug}

### 2. GitHub Repository
- Creator publishes their product as a GitHub repo
- README carries MyClaude badge + install link
- Topics: `claude-code`, `agent-skills`, `myclaude`, `{product-type}`
- This feeds awesome-claude-code lists and GitHub search

### 3. Anthropic Plugin Marketplace
- plugin.json generated by `/package`
- Submit via `/plugin marketplace add` in Claude Code
- Homepage links back to myclaude.sh

### 4. Agent Skills Registry (33+ platforms)
- agentskills.yaml generated by `/package`
- Compatible with Cursor, Codex, Gemini CLI, Copilot, Kiro, etc.
- marketplace field links to myclaude.sh product page

### 5. npm / Package Registries
- For creators who want programmatic distribution
- package.json can reference myclaude.sh as homepage
- README carries same badges and attribution

### 6. Community Channels
- Reddit: r/ClaudeCode, r/ClaudeAI
- X/Twitter: #ClaudeCode, #MyClaude
- Dev.to, Hashnode articles
- Discord communities

## Signature Elements (injected by /package)

### README Footer (mandatory)
Every product README ends with a distribution footer containing:
- MCS quality badge (links to MyClaude quality spec)
- "Available on MyClaude" badge (links to product page)
- Install command (one-liner)
- Agent Skills compatibility note

### Manifest Metadata
- vault.yaml: `engine: myclaude-studio-engine` field
- plugin.json: `homepage` pointing to myclaude.sh/p/{slug}
- agentskills.yaml: `marketplace` pointing to myclaude.sh/p/{slug}

### Attribution Comment (in primary product file)
A single comment line at the end of SKILL.md / AGENT.md / etc:
```
<!-- Published on MyClaude (myclaude.sh) | Quality: MCS-{level} -->
```
This is invisible to Claude but visible in source, creating indexable references.

## Advanced: 5 Layers of Viral Distribution DNA

### Layer 1: Frontmatter as Distribution Channel (AI-level viral)
The `description` field in SKILL.md frontmatter includes `myclaude.sh/p/{slug}`.
Claude Code reads this field in EVERY session of EVERY buyer. When a user asks
"what skills do I have?", Claude surfaces the link. **The AI itself becomes the
distribution vector.** No other marketplace has this — it's unique to how Claude
Code loads skill descriptions into context.

### Layer 2: Cross-Product Discovery Graph
Every product's Composability section links to `myclaude.sh/explore?category={cat}`.
When a buyer reads one product, they discover related products. This creates an
internal link graph that strengthens ALL products' discoverability and drives
repeat purchases.

### Layer 3: Attribution Comment (Source-level indexing)
```html
<!-- Published on MyClaude (myclaude.sh) | Quality: MCS-{level} ({score}%) | Engine: Studio v2 -->
```
Invisible to Claude, visible in raw source on GitHub, indexable by search engines.
Every product file on every GitHub repo in the ecosystem carries this reference.

### Layer 4: Triple Manifest Backlinks
- vault.yaml → `marketplace: "https://myclaude.sh/p/{slug}"`
- plugin.json → `homepage: "https://myclaude.sh/p/{slug}"`
- agentskills.yaml → `marketplace: "https://myclaude.sh/p/{slug}"`
Three manifest formats, three ecosystems, all pointing home.

### Layer 5: README Badge Stack
```markdown
[![MCS 2](https://myclaude.sh/badge/mcs/2.svg)](https://myclaude.sh/quality)
[![Available on MyClaude](https://myclaude.sh/badge/available.svg)](https://myclaude.sh/p/{slug})
<sub>Quality-validated by MyClaude Studio Engine | Browse marketplace</sub>
```
Visual, clickable, on every GitHub README. Two backlinks per product.
Badges become a recognized quality signal — creators WANT them displayed.
