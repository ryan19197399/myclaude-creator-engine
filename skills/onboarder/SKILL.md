# Onboarder

Discover and persist creator profile through guided conversation. Build `creator.yaml`.

**When to use:** First time using the Creator Engine in a project, or when profile is stale and needs update.

**When NOT to use:** When `creator.yaml` already exists and is accurate — use `/update-profile` instead of re-running full onboarding.

---

## Activation Protocol

1. Check for `creator.yaml` in project root
2. If it **exists**: load it, display summary, ask "Is this still accurate? (y/n)"
   - If yes → exit, profile is current
   - If no → enter UPDATE mode (skip welcome, re-ask changed sections only)
3. If it **does not exist**: enter FULL ONBOARDING mode

---

## Core Instructions

### FULL ONBOARDING FLOW

**Step 1 — Welcome + Language Detection**

Detect language from system locale. If ambiguous, ask:

```
Welcome to MyClaude Creator Engine!
I'll help you build your creator profile (takes ~3 minutes).
What language should I use? [en / pt-BR / es / other]
```

Proceed in the detected/selected language for all subsequent questions.

**Step 2 — Name + Username**

```
What is your name? (as you'd like it shown on marketplace)
What is your MyClaude username? (e.g., @yourhandle — leave blank if not registered yet)
```

**Step 3 — Expertise Domains**

```
What topics do you know deeply? List your areas of expertise.
Examples: "Python automation", "B2B sales copywriting", "financial modeling", "Claude Code extensions"
(Enter 1-5 areas, one per line or comma-separated)
```

**Step 4 — Goals**

```
What do you want to do with MyClaude? (select all that apply)
  1. Sell products for income
  2. Share free tools with the community
  3. Build a portfolio to showcase expertise
  4. Automate my own workflows (personal use)
  5. Other: ___
```

**Step 5 — Technical Level**

```
How would you describe your technical level?
  1. Beginner — I'm new to Claude Code and prompt engineering
  2. Intermediate — I write prompts and use Claude Code regularly
  3. Advanced — I build skills, agents, and complex workflows
  4. Expert — I architect systems and understand Claude's internals
```

**Step 6 — Profile Type Classification**

Infer from answers. Do not ask directly. Classification rules:
- Has coding expertise + advanced/expert → `developer`
- Strong domain knowledge + no coding → `domain-expert`
- Marketing/sales goals + copywriting skills → `marketer`
- Heavy prompt work, intermediate level → `prompt-engineer`
- Spans multiple categories → `hybrid`

**Step 7 — Environment Scan**

Silently scan local environment:
- Check `.claude/skills/` — count installed skills
- Check `.claude/commands/` — count slash commands
- Check `workspace/` — count product drafts
- Note any existing products that could be published

Report findings:
```
Environment scan complete:
  - Installed skills: 3
  - Slash commands: 7
  - Drafts in workspace/: 2
```

**Step 8 — Preferences**

```
Default license for your products? [MIT / Apache-2.0 / Proprietary / other] (default: MIT)
Default product category? [skills / agents / prompts / workflows / other] (default: skills)
Quality target? [MCS-1 Publishable / MCS-2 Quality / MCS-3 State-of-the-Art] (default: MCS-2)
Pricing strategy? [free / freemium / premium] (default: free)
```

**Step 9 — Generate creator.yaml**

Write file to project root. Schema:

```yaml
# Auto-generated by /onboard — do not edit manually
# Update via: /onboard (run again to update)
creator:
  name: "{name}"
  myclaude_username: "{@username or null}"
  language: "{detected language code}"
  onboarded_at: "{YYYY-MM-DD}"

  profile:
    type: "{developer|prompt-engineer|domain-expert|marketer|hybrid}"
    expertise:
      - "{domain 1}"
      - "{domain 2}"
    technical_level: "{beginner|intermediate|advanced|expert}"
    goals:
      - "{goal 1}"
      - "{goal 2}"

  preferences:
    default_license: "{license}"
    default_category: "{category}"
    pricing_strategy: "{free|freemium|premium}"
    quality_target: "{MCS-1|MCS-2|MCS-3}"

  inventory:
    local_skills: {count from scan}
    local_agents: {count from scan}
    published_products: 0
    unpublished_drafts: {count from scan}

  recommendations:
    suggested_categories: ["{category 1}", "{category 2}"]
    market_gaps_matched: []
```

**Step 10 — Persona-Adaptive Recommendations**

Show next steps tailored to profile type:

| Profile Type | Recommended First Action |
|---|---|
| `developer` | `/create skill` — build a tool around your code expertise |
| `domain-expert` | `/create prompt` or `/scan-market` — monetize your domain knowledge |
| `marketer` | `/scan-market` — find gaps that match your audience |
| `prompt-engineer` | `/create prompt` or `/create agent` — start with what you know |
| `hybrid` | `/create system` — combine multiple product types |

```
Profile created! Here's what I recommend next:

  [persona-specific recommendation from table above]

  Other options:
  - /create — start building any product type
  - /scan-market — see what's selling in the marketplace
  - /validate — check a product you've already built
```

### UPDATE MODE FLOW

Load existing `creator.yaml`. Show current values. Ask only:

```
Which sections need updating?
  1. Expertise domains
  2. Goals
  3. Technical level
  4. Preferences (license, category, quality target)
  5. All of the above
```

Re-run only selected sections. Preserve `onboarded_at`. Update `updated_at` field.

```
Profile saved to creator.yaml
[persona-adaptive next step recommendations]
```

---

## Quality Gate

- `creator.yaml` must be valid YAML (no syntax errors)
- All required fields must be non-empty: `name`, `language`, `profile.type`, `profile.technical_level`, `preferences.default_license`, `preferences.default_category`, `preferences.quality_target`
- Profile type must be one of: `developer`, `prompt-engineer`, `domain-expert`, `marketer`, `hybrid`
- Technical level must be one of: `beginner`, `intermediate`, `advanced`, `expert`
- License must be from CE-D40 approved list: MIT, Apache-2.0, GPL-3.0, BSD-3-Clause, ISC, CC-BY-4.0, CC-BY-SA-4.0, CC0-1.0, Proprietary, Custom

---

## Decision Notes

**CE-D11:** Profile is generated once via `/onboard`, then updated automatically as the creator publishes products and receives feedback. Downstream skills (Scaffolder, Publisher) read `creator.yaml` for defaults — do not prompt for information already captured here.

**Why not ask profile type directly:** Creators often don't know which category they fit. Classification from behavior is more accurate than self-report.
